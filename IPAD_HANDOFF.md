# Reporting Room — iPad App Handoff Checklist

This document is the step-by-step guide your developer or consultant needs to
build, sign, and submit the Reporting Room iPad app to the Apple App Store.
Replit cannot build or sign iOS apps, so these steps are completed on a Mac.

---

## Prerequisites

| Requirement | Notes |
|---|---|
| Mac running macOS 14 (Sonoma) or later | Xcode 15+ requires it |
| Xcode 15+ installed | Free from the Mac App Store |
| Apple Developer Program membership | USD $99/year — enroll at developer.apple.com |
| Node.js 20+ installed on the Mac | Install via https://nodejs.org |
| The Replit project cloned locally | `git clone <your-repo-url>` |

---

## Step 1 — Clone & install dependencies

```bash
git clone <your-repo-url> reporting-room
cd reporting-room
npm install
```

---

## Step 2 — Set the production backend URL

The iOS app must know where the live backend lives.  Create a `.env.native`
file (or set the variable in your CI/CD system) with the deployed Replit URL:

```bash
# .env.native (do NOT commit this file)
VITE_API_BASE_URL=https://reporting-room.<your-username>.replit.app
```

Build the frontend with this variable:

```bash
VITE_API_BASE_URL=https://reporting-room.<your-username>.replit.app \
  npm run build
```

The compiled output lands in `dist/public/` — this is what the iPad app
serves locally, with all API calls going to the backend URL above.

---

## Step 3 — Install CocoaPods dependencies (first time only)

The `ios/` Xcode project is already committed to the repository.  You do **not**
need to run `npx cap add ios` again.  However, the native CocoaPods dependencies
must be installed on your Mac the first time you clone the project.

```bash
# Install CocoaPods if you don't have it
sudo gem install cocoapods

# Install the iOS dependencies
cd ios/App
pod install
cd ../..
```

After `pod install` completes, always open the **workspace** file, not the project file:
```
ios/App/App.xcworkspace
```

---

## Step 4 — Sync the web build into the iOS project

Every time you rebuild the frontend, push the new files into the native shell:

```bash
npx cap sync ios
```

---

## Step 5 — Open Xcode

```bash
npx cap open ios
```

Xcode will open.  **Always use the workspace file** (`ios/App/App.xcworkspace`),
not the project file — CocoaPods-linked projects only build correctly from the
workspace.

---

## Step 6 — Configure signing & capabilities

1. In Xcode, select the **App** target in the project navigator.
2. Open the **Signing & Capabilities** tab.
3. Under **Team**, select your Apple Developer team.
4. Xcode will generate a provisioning profile automatically.
5. Verify the **Bundle Identifier** matches `au.com.reportingroom.app`
   (edit it here if you need a different ID — it must be unique on the
   App Store).

---

## Step 7 — Set the display name & version

In `ios/App/App/Info.plist`, confirm/update:

| Key | Value |
|---|---|
| `CFBundleDisplayName` | `Reporting Room` |
| `CFBundleShortVersionString` | `1.0.0` (user-visible version) |
| `CFBundleVersion` | `1` (build number — increment each submission) |

---

## Step 8 — Add required Privacy Usage Descriptions

Apple requires a human-readable explanation for every permission the app uses.
Add these keys to `ios/App/App/Info.plist` (Xcode also has a UI for this under
Info → Custom iOS Target Properties):

| Key | Suggested value |
|---|---|
| `NSCameraUsageDescription` | `Reporting Room uses the camera to capture ultrasound worksheet images for report generation.` |
| `NSPhotoLibraryUsageDescription` | `Reporting Room accesses your photo library to attach worksheet images to reports.` |
| `NSMicrophoneUsageDescription` | `Reporting Room uses the microphone for voice dictation in consultation notes.` |
| `NSPhotoLibraryAddUsageDescription` | `Reporting Room saves exported report PDFs to your photo library.` |

Only include descriptions for features you actually enable.  If dictation is
not yet live, omit the microphone entry to avoid App Review questions.

---

## Step 9 — App icon & launch screen

### App Icon
Replace the placeholder icons in:
```
ios/App/App/Assets.xcassets/AppIcon.appiconset/
```
Apple requires a single 1024×1024 PNG (no alpha) for App Store submission.
All other sizes are generated by Xcode or you can use a tool like
[IconKitchen](https://icon.kitchen) to generate the full set.

### Launch Screen (Splash)
The default Capacitor splash screen shows the app icon on a white background.
To customise it, edit `ios/App/App/LaunchScreen.storyboard` in Xcode.

---

## Step 10 — Build & run on a device or simulator

### Simulator (no Apple Developer account needed for this step)
1. In Xcode, select an **iPad** simulator from the device picker.
2. Press **⌘R** (Run).

### Real iPad
1. Connect the iPad via USB and trust the Mac.
2. Select the iPad in the device picker.
3. Press **⌘R**.
4. On the iPad, go to **Settings → General → VPN & Device Management** and
   trust your developer certificate.

---

## Step 11 — App Store submission

1. In Xcode, choose **Product → Archive**.
2. Once the archive finishes, the **Organizer** window opens.
3. Click **Distribute App → App Store Connect → Upload**.
4. Log in to [App Store Connect](https://appstoreconnect.apple.com).
5. Under **My Apps**, create a new app (or find the existing one).
6. Fill in metadata: description, keywords, screenshots (iPad 12.9" required),
   support URL, privacy policy URL.
7. Select the build you uploaded and submit for review.

Apple review typically takes 1–3 business days.

## Troubleshooting

### "Session cookie not sent" / user is logged out after every request
- Confirm the deployed backend URL in `VITE_API_BASE_URL` uses **HTTPS**.
- The session cookie is `secure: true; sameSite: None` — it requires HTTPS.
- In Xcode, check the device's network log (Debug → Attach to Process) or use
  Charles Proxy to inspect the requests.

### CORS error in the WebView console
- The backend `CORS_ORIGIN` environment variable (on Replit) must include
  `capacitor://localhost` and `https://localhost`.
  - Go to your Replit project → **Secrets** → add/update `CORS_ORIGIN` with a
    comma-separated list:
    ```
    capacitor://localhost,https://localhost,https://reporting-room.<user>.replit.app
    ```
  - Redeploy the backend after updating secrets.

### White screen on launch
- Make sure you ran `npm run build` **then** `npx cap sync` in that order.
- Open the Safari Web Inspector (Safari → Develop → [your device]) and check
  the JavaScript console for errors.

### Camera / photo picker not working
- Verify the `NSCameraUsageDescription` key is in `Info.plist`.
- On simulator, camera is not available — test on a real device.

---

## Environment variables summary

| Variable | Where set | Purpose |
|---|---|---|
| `VITE_API_BASE_URL` | Build-time (`.env.native` or CI) | Points native app to deployed backend |
| `CORS_ORIGIN` | Replit Secrets (server-side) | Adds native app origins to CORS allowlist |
| `SESSION_SECRET` | Replit Secrets (server-side) | Already configured — do not change |

---

## Keeping the app up to date

After any code changes to the Reporting Room web app:

```bash
# 1. Build the new web bundle
VITE_API_BASE_URL=https://reporting-room.<user>.replit.app npm run build

# 2. Sync into the iOS project
npx cap sync ios

# 3. Open Xcode, increment CFBundleVersion, archive and upload
npx cap open ios
```
